PREQLY BACKEND: HOW TO RUN

1. Go into the backend folder:

cd /Users/jamesnguyen/AllProjects/preqly/backend

2. Create a virtual environment if you do not already have one:

python3 -m venv .venv

3. Activate it:

source .venv/bin/activate

4. Install dependencies:

pip install -r requirements.txt

5. Make sure your API keys exist.

The backend now uses:
- `YOUTUBE_API_KEY` for `/api/teach-agent`
- `GEMINI_API_KEY` for `/api/flashcard-agent`
- `GEMINI_API_KEY` for `/api/test-agent`

You can place them in either:
- `/Users/jamesnguyen/AllProjects/preqly/backend/.env`
- `/Users/jamesnguyen/AllProjects/preqly/.env`

Example:

YOUTUBE_API_KEY="your_youtube_key"
GEMINI_API_KEY="your_gemini_key"
GEMINI_MODEL="gemini-3-flash-preview"

6. Start the backend:

uvicorn app.main:app --reload

7. Open the Swagger docs:

http://127.0.0.1:8000/docs


LOCAL BASE URL

http://127.0.0.1:8000


AVAILABLE ENDPOINTS

1. GET /health
2. POST /api/teach-agent
3. POST /api/flashcard-agent
4. POST /api/flashcard-agent/review
5. POST /api/test-agent


1) HEALTH CHECK

Request:

GET /health

Response:

{
  "status": "ok"
}

- takes a topic name
- searches YouTube

4) FLASHCARD REVIEW

What it does:
- updates the review schedule for one flashcard

Endpoint:

POST /api/flashcard-agent/review

Correct input JSON:

{
  "schedule": {
    "box": 1,
    "interval_days": 1,
    "ease_factor": 2.5,
    "due_at": "2026-03-29T17:47:30.758876Z",
    "last_reviewed_at": null
  },
  "rating": "good"
}

Allowed rating values:
- `again`
- `hard`
- `good`
- `easy`

Correct output shape:

{
  "updated_schedule": {
    "box": 2,
    "interval_days": 2,
    "ease_factor": 2.5,
    "due_at": "2026-03-30T17:47:30.758876Z",
    "last_reviewed_at": "2026-03-28T17:47:30.758876Z"
  }
}

uvicorn app.main:app --reload

2. Open:

http://127.0.0.1:8000/docs

3. Test `/api/teach-agent` with:

{
  "topic_name": "Binary Search",
  "max_results": 5
}

4. Copy the first 3 videos from the result

5. Paste those 3 videos into `/api/flashcard-agent`

6. Paste those same 3 videos into `/api/test-agent`


COMMON MISTAKES

- using `vid_id` instead of `video_id`
- pasting JavaScript object syntax instead of valid JSON
- leaving trailing commas in JSON
- passing fewer than 3 videos or more than 3 videos
- missing `GEMINI_API_KEY`
- using private or unlisted videos with Gemini
